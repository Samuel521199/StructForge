## 文档阅读建议

### 对于架构师/技术负责人
1. 先阅读 [架构设计文档](architecture.md) 了解整体架构
2. 阅读 [设计分析文档](design-analysis.md) 了解设计决策
3. 参考 [项目结构文档](project-structure.md) 了解实现细节

### 对于开发人员
1. 先阅读 [项目结构文档](project-structure.md) 了解代码组织
2. 阅读 [架构设计文档](architecture.md) 了解服务职责
3. 参考 [设计分析文档](design-analysis.md) 了解实现细节

### 对于新加入的成员
1. 按顺序阅读所有文档
2. 重点关注自己负责的模块
3. 参考代码结构理解实现
